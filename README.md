# neo4j-mcp-workspace-template

A template for end-to-end Neo4j graph development using MCP servers. Compatible with **Cursor** and **Claude Code**.

## Quick Start

```bash
git clone https://github.com/neo4j-field/neo4j-mcp-workspace-template
cd neo4j-mcp-workspace-template
chmod +x setup.sh && ./setup.sh
```

Then open the IDE:

- **Cursor** ‚Äî open this folder, then run the `@setup-workspace` skill to validate
- **Claude Code** ‚Äî run `claude` from this directory, then `/setup-workspace` to validate

`setup.sh` will:
1. Prompt for your Neo4j credentials and OpenAI API key
2. Write `.env` with your configuration
3. Install dependencies for the 3 local MCP servers
4. Generate `.cursor/mcp.json` (Cursor) and `.mcp.json` (Claude Code) with absolute paths and literal credential values

## Requirements

- **Python 3.10+**
- **[uv](https://docs.astral.sh/uv/getting-started/installation/)** ‚Äî fast Python package manager
- **Neo4j 2025.01+** ‚Äî required for native `VECTOR` type used by lexical-graph
  - [Neo4j Desktop](https://neo4j.com/download/) or [AuraDB](https://neo4j.com/cloud/platform/aura-graph-database/)
- **LLM API key** ‚Äî for embedding generation and entity extraction (OpenAI or Anthropic)
- **Google MCP Toolbox** (optional) ‚Äî only if using BigQuery as a data source

## MCP Servers

This workspace configures six MCP servers:

### Data Modeling
Designs and validates graph schemas from sample data. Provides example data models and Mermaid visualizations.

- **Server:** `mcp-neo4j-data-modeling` (remote via `uvx`)
- **Required credentials:** none

### Cypher
Executes read and write Cypher queries against Neo4j.

- **Server:** `mcp-neo4j-cypher` (remote via `uvx`)
- **Required credentials:** `NEO4J_URI`, `NEO4J_USERNAME`, `NEO4J_PASSWORD`, `NEO4J_DATABASE`

### Ingest
Loads structured CSV data into Neo4j using parameterized Cypher queries generated by the Data Modeling server.

- **Server:** `mcp-neo4j-ingest` (local ‚Äî `mcp-neo4j-ingest/`)
- **Required credentials:** Neo4j connection vars

### Lexical Graph
Parses PDFs into a searchable graph with `Document` and `Chunk` nodes, vector embeddings, and fulltext indexes. Supports 4 parse modes:
- `pymupdf` ‚Äî fast text extraction (default)
- `docling` ‚Äî structural layout detection
- `page_image` ‚Äî vision model per page
- `vlm_blocks` ‚Äî vision model per layout block (highest quality)

- **Server:** `mcp-neo4j-lexical-graph` (local ‚Äî `mcp-neo4j-lexical-graph/`)
- **Required credentials:** Neo4j connection vars + LLM API key + `EMBEDDING_MODEL`

### Entity Graph
Extracts structured entities from lexical graph chunks using an LLM. Supports 100+ providers via LiteLLM. Processing runs asynchronously in the background.

- **Server:** `mcp-neo4j-entity-graph` (local ‚Äî `mcp-neo4j-entity-graph/`)
- **Required credentials:** Neo4j connection vars + LLM API key + `EXTRACTION_MODEL`

### GraphRAG
Retrieval layer for RAG applications: vector search, fulltext search, and graph-traversal queries against the Neo4j graph. Use after the lexical graph and entity graph are populated.

- **Server:** `mcp-neo4j-graphrag` (remote via `uvx`)
- **Key tools:** `get_neo4j_schema_and_indexes`, `vector_search`, `fulltext_search`, `read_neo4j_cypher`, `search_cypher_query`
- **Required credentials:** Neo4j connection vars + `OPENAI_API_KEY` + `EMBEDDING_MODEL`

### BigQuery (optional)
Connects to BigQuery as a source database. Requires the [Google MCP Toolbox](https://github.com/googleapis/genai-toolbox) CLI.

```bash
brew install mcp-toolbox
```

## Workflow

### Structured data (CSV)

1. **Discovery** ‚Äî analyze CSV samples; identify entities, relationships, and use cases
2. **Model** ‚Äî design a graph data model with the Data Modeling server
3. **Ingest** ‚Äî load CSV data into Neo4j with the Ingest server
4. **Query** ‚Äî generate Cypher for each use case
5. **Validate** ‚Äî run queries and verify results address the use cases

### Unstructured data (PDF)

1. **Discovery** ‚Äî review PDFs; identify entities and use cases
2. **Lexical Graph** ‚Äî parse PDFs into chunks with the Lexical Graph server
3. **Embed** ‚Äî generate vector embeddings for semantic search
4. **Entity Graph** ‚Äî extract entities from chunks with the Entity Graph server
5. **Model** ‚Äî design a graph data model based on extracted entities
6. **Query** ‚Äî generate Cypher for each use case
7. **Validate** ‚Äî run queries and verify results

Run `/develop-neo4j-graph` (Cursor skill or Claude Code slash command) for the full guided workflow.

## Setup

### Primary: `./setup.sh`

```bash
chmod +x setup.sh && ./setup.sh
```

The script is idempotent. Re-run it any time to:
- Regenerate `.cursor/mcp.json` and `.mcp.json` after credential changes
- Re-install server dependencies after a `git pull`

### Alternative: Manual Setup

<details>
<summary>Manual setup steps</summary>

**1. Create `.env`**

Copy `.env.example` to `.env` and fill in your values:
```bash
cp .env.example .env
# Edit .env with your Neo4j credentials and API keys
```

**2. Install local server dependencies**

```bash
uv sync --directory mcp-neo4j-ingest
uv sync --directory mcp-neo4j-lexical-graph
uv sync --directory mcp-neo4j-entity-graph
```

**3. Create `.cursor/mcp.json`**

Copy `.cursor/mcp.json.example` to `.cursor/mcp.json` and fill in your credentials:
```bash
cp .cursor/mcp.json.example .cursor/mcp.json
# Edit .cursor/mcp.json ‚Äî replace placeholder values with real credentials
```

Important: replace the relative `--directory mcp-neo4j-ingest` paths with absolute paths, or `setup.sh` handles this automatically.

**4. For Claude Code, create `.mcp.json`**

Copy `.cursor/mcp.json` to `.mcp.json` at the repo root (same content):
```bash
cp .cursor/mcp.json .mcp.json
```

</details>
